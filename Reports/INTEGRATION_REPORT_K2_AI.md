### 2.4 K2Provider Architecture
The `K2Provider` class was implemented with high-reliability patterns:
- **Resilience:** 3-attempt exponential backoff for network/5xx errors and `retry-after` header support for 429 rate limits.
- **Timeouts:** Fixed 90-second timeout using `AbortSignal.timeout`.
- **Clean Output:** Automated stripping of `<think>` XML reasoning blocks to provide clean Markdown content to the UI.
- **Truncation:** Smart truncation at ~12,000 characters (sentence-aware) to preserve model context.

### 2.5 Async Worker (JobWorker)
- **Pipeline:** Logic updated to prioritize `processedContent` (cleaned text) over `rawContent`.
- **Idempotency:** Replaced creation logic with `upsert` to allow seamless job re-runs without data duplication.
- **Graceful Failure:** Unimplemented job types (Embed/Quiz) are marked as `FAILED` with specific messages rather than crashing the loop.
